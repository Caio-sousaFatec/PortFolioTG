## Problema Proposto

A empresa possui um sistema de processamento de dados em cascata chamado pipeline, que consiste em vários estágios. Este sistema consiste em vários estágios que são coordenados automaticamente com base nas características das fontes de dados e nas necessidades dos clientes em termos de algoritmos de IA ou modelos matemáticos. Na metodologia de implementação do sistema, é necessário configurar as fontes de dados envolvidas para que a plataforma funcione corretamente. No entanto, a configuração manual das fontes de dados é feita manualmente, sendo um ponto crítico que consome muito tempo dos técnicos e cria uma dependência excessiva de especialistas para realizar tal tarefa.

---

## Solução apresentada

---

Foi desenvolvida uma **aplicação Web completa**, permitindo:

- Cadastro e gerenciamento de clientes, soluções e usuários.  
- Upload de dados com **validação automática** de estrutura.  
- Aplicação de regras de negócios via mapeamento de campos.  
- Visualização de métricas em um **dashboard analítico**.  
- Organização de pipeline configurável e rastreável.  
- Fluxo orientado para garantir consistência dos dados enviados ao sistema.

### **Requisitos Funcionais Concluídos**

- Cadastrar e gerenciar clientes e usuários.  
- Realizar upload de arquivos para validação.  
- Aplicar e revisar regras de negócios (“de/para”).  
- Exibir dashboards analíticos com métricas das configurações.  
- Permitir acompanhamento visual das etapas da pipeline.  
- Validar tipos, campos e estrutura antes de continuar o fluxo.

### **Requisitos Não Funcionais**

- Garantir segurança no acesso e manipulação de dados.  
- Alta disponibilidade e confiabilidade do ambiente.  
- Interface intuitiva e acessível para diferentes perfis de usuário.  
- Manutenibilidade do código-fonte (arquitetura organizada).  
- Logs e rastreabilidade durante o fluxo da pipeline.
